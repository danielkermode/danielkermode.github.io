Javascript Basics

Wednesday, 2016-04-06

Javascript vs HTML and CSS

Javascript, HTML and CSS must all be understood in the context of the internet browser. The first internet browser as far as I can tell was Mosaic, and shortly after this came Netscape and Internet Explorer (the browser everyone now loves to hate). Javascript was created in 10 days by Brendan Eich, who was working at Netscape at the time. The original name was Mocha, which then became LiveScript. A trademark license from Sun meant the language was eventually called "Javascript" (after Sun's programming language Java) which proved to be a great marketing move.

So why do we need these three different languages to develop webpages in a browser? To confuse people? Maybe. Each serve a very different purpose, but together they give the advantage of making rich, interactive web pages relatively easy to develop.
HTML by itself creates the structure of a page (normally a page in a web browser), and the code itself consists of predefined "tags" such as <div> or <header>.
CSS defines the style of a page with predefined properties of particular HTML elements. The key thing to note is that both HTML and CSS are very limited in terms of how they can interact with a webpage. This doesn't mean they aren't useful, as they provide a very high-level abstraction to lay out a document. However, if you would like to design, for example, a simple interactive computer game in a browser (that isn't just a text adventure) you are going to find HTML and CSS lacking.

This is where Javascript comes in handy. Javascript is a fully-fledged programming language (some may disagree) that uses lower-level programming concepts such as functions and arrays as well as interaction with the DOM to provide a more granular level of control to the developer. Initially Javascript didn't have access to the DOM and was mainly used for interacting with forms, giving feedback to the user, and detecting when they do certain things. Functions such as alert(), confirm() and prompt() could (and still can) be used to make windows pop up in the browser that do simple tasks.


With browser implementation of the DOM, Javascript could use an object representation of the document to adjust it instead of just making windows pop up. Control Flow and Loops

Javascript, like many other programming languages, uses some keywords that describe general actions for the computer to complete, and general logic for completing these actions. Taking the "if" keyword as an example: if I am fat, I go to the gym. It mirrors English, although computer code has to be far more exact (How fat? What is "fat" exactly? Who am "I"?). Statements like these form the "control flow" of a Javascript program.

Loops are part of the control flow, and have keywords that describe repetitive actions. Taking the "while" keyword as an example: while I am thin, I eat cakes. Again precision is important for a computer (How thin? What type of cake?). Loops are extremely useful since you don't need to repeat identical lines of code. In this example, you wouldn't have to explicitly tell the computer "Eat a cake. Eat a cake. Eat a cake. Done, I'm no longer thin."

Arrays and Objects

Objects store forms of data (numbers, objects, functions etc.) in Javascript as properties. Example syntax: 

          
          var obj = {
            property: 1,
            property1: "string",
            property2: {},
            property3: function func() {
              //do something
            }
          };
These properties can be accessed two ways.
Dot notation: obj.property
Bracket notation: obj["property"]

Arrays in Javascript are actually just objects but they have special "index" properties which are all numbers, and they have other functionality such as length (which basically gives the last valid index + 1 of the array). Literals use square brackets instead of curly ones, and you don't need to specify property names (since they will be numbers). Example syntax: 

          
          var arr = [1, "string", {}, function func(){}]
Now you can access these properties with bracket notation and the desired index (starting from 0).
This gets the second value ("string"): arr[1]

Of course, these are just the basics of arrays and objects. For more details MDN has excellent documentation.

Functions

Functions in Javascript are actually still objects. They are "function" objects with special properties and methods. From MDN docs:

Generally speaking, a function is a "subprogram" that can be called by code external (or internal in the case of recursion) to the function.

Conceptually, functions are just a way to bracket sets of commands for the computer. Perhaps makeBed() could be a function. Then everytime you "call" this function, it will make your bed (this could be tricky to actually program). Or maybe you design a function cookMeal(). Then if you tell the computer "cookMeal()!" it will hopefully cook a meal for you.

They are a useful concept because if you craft a well-thought-out function, you can call it many times to make your life easy. Taking the cookMeal() function above, you can simply ask the computer to run your function every evening when it's time for dinner. You can also pass in "arguments" to a function which will adjust what it does. For example, the meal function may take an argument "food". If the user prefers pasta, they can run cookMeal("pasta") and buon appetito! You can even pass in functions as arguments to another function (functionception).

An important feature of functions is that they must always return a value. The default is undefined, but this can be overridden with a return statement. Extending the example above, this could be the end of the function: return cookedFood;.
