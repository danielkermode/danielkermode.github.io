What is a responsive site?

A responsive site is one that changes depending on the device/screen size used. Responsiveness is important because as I explained in the previous blog, some users of a website will probably be using a phone or something similar, and it's important that they don't get deterred by a site that clearly hasn't been made to suit their smaller-screen needs. I adjusted the nav bar of this site to be more responsive so that if you're on mobile or another small device, you should see that button with three lines that's on a bunch of mobile sites (if you're on a larger screen, you can resize the window to see it). It will let you view the links in a more controlled fashion.

Why mobile first?

Mobile-first queries are how Skeleton's grid is built (Skeleton is a lightweight CSS framework) and is the preferrable method of organizing CSS. It means all styles outside of a query apply to all devices, then larger devices are targeted for enhancement. This prevents small devices from having to parse tons of unused CSS.

Frameworks

"Framework" is a broad term, but generally a framework in software is understood to mean an abstraction tool that makes it easier to develop applications. For web development, web frameworks are used.

This article is just one of many explaining the pros and cons of web frameworks. It was the first that came up in a google search, so it's probably valid. Here's a summary of the good points this individual makes:

    Pros

  • Efficiency. Pre-built functionality.
  • Security. A popular framework will prioritise security (hopefully).
  • Cost. Most are free.
  • Support. Widely used normally means lots of documentation and people who can help.

    Cons

  • You learn the framework, not the language. There is the danger that you'll rely on it, and not understand the fundamentals of what you are doing.
  • Limitation. You'll have to follow the framework's core behaviour.
  • Public. Ironically security is both a strength and a weakness, since public code also means nothing is hidden from anyone.

Wireframes

Wireframes are used in design to help scaffold out the basic layout and structure of a page. They are a visual guide for a developer, but can be created by anyone, eg. a visual designer. They help to show roughly what the user experience of the website will be.
