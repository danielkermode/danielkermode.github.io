JS language

Thursday, 2016-04-21

Javascript derives its syntax from Java which in turn derives some of its syntax from C.

Parentheses

Parentheses () are used in a number of cases:

  • 1. Invoking functions/methods, eg. console.log('hi');.
  • 2. Adjusting the order of evaluation for certain Javascript expressions, eg.
    var num = (valIsTrue? 2 : 3) * 3;. They can work in a similar fashion to parentheses in mathematics, and will prioritise whatever is in the parentheses for evaluation. So in the example, if the variable "valIsTrue" is true, then "num" will evaluate to (2) * 3 = 6, whereas without parentheses the code will just evaluate to 2.
  • 3. After certain keywords that define control flow (such as "if", "for", "while", "switch"), eg.
    if(blah > 11) console.log('blah');. The code in the parentheses in this case is an expression that determines whether the code after the "if" statement should be executed.
  • 4. Creating an instance of an object with "new", eg. new Number(3);. "Number" is a function that will return a special number object if called with the preceding keyword "new". Used this way, it's called a "constructor".
  • 5. An anonymous function (one not assigned to any variable) can be created by wrapping a function expression with parentheses, eg.
    (function(){return 'val'});. You can then instantly invoke the function by adding parentheses after the last curly bracket:
    (function(){return 'val'}()); or after the expression:
    (function(){return 'val'})();. Both do the same thing. These are useful for scoping variables within the function ("scope" is the environment in which the variable can be accessed).

Brackets

Square brackets [] are mainly used when dealing with arrays. Array literals are created with square brackets, eg.
var arr = [['hi'], 1, 2, 3, 'let\'s go'];
Bracket notation can be used to access an element of this array with a number, eg. arr[arr.length-1] gets the last index "let's go". Bracket notation can also be used for any objects, eg. anObject['aProperty'] which must be accessed with a string as the property value.

Braces

Braces {}, informally known as curly brackets, are similar to parentheses but have slightly different use cases:

  • 1. Function bodies, eg. function foodBar(){return 'val';}; where the return statement is the body here.
  • 2. Loop/conditional bodies, eg. if(true) {console.log('true');} where the "console.log" is the body of the "if" statement in this case. If the body is just one statement, the curly brackets can be left out. However multi-statement bodies need the curly brackets.
  • 3. Object literals, eg.
    var obj = {};.
  • 4. You can also place them haphazardly in your code, eg. 
            
        var ugly = 'code';
        {
          console.log(ugly);
        }
    This will work, but is pointless in most current Javascript interpreters. Newer versions of Javascript engines may support block-scoping variables, making this useful.

Single and Double Quotes

Single quotes var s = 'string'; and double quotes var s = "string"; are both used for string literals. They can also be used for object property names, eg. var obj = {'a-property-with-dashes': 1};. Both do the same thing, and usage comes down to stylistic preference. The important thing is to remain consistent, and also to keep in mind that you will have to escape the quote character you use within strings. The JS escape character is the backslash. For example, if you'd like to create a string containing a quote using double quotes:
var s = "this is \"quoteception\"!";
