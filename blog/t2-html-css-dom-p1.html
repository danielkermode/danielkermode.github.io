Analogy for HTML, CSS and the DOM

The analogy used in the Udacity tutorial was a house for the HTML, and decorations for CSS. CSS is 'style' for your HTML (it's in the name: Cascading Style Sheets!) whereas HTML determines the actual structure of your page, and most importantly the view layout. As for the DOM, according to the Mozilla Developer Network, it's an interface for HTML and XML documents. This just means it's a way to represent the data contained in a document (and for the web, that means web pages). The DOM can be accessed by programming languages, and most commonly Javascript is used for this. Without the DOM, Javascript code couldn't manipulate a web page in the way it does. Another thing that MDN notes is that browsers implement functionality of the DOM (eg. Chrome) but the DOM might differ between different browsers. That's not something to lose sleep over though! The main point is that the structure of a web page consists of a DOM, and you can use this DOM through code to build good-looking web pages.

Boxifying Design

Because HTML is represented in terms of boxes, when you think about laying out designs for web pages, it's best to think in terms of boxes. Even elements of your page that aren't square can be thought of as boxes by enclosing the object in a box. Then you can fit these in with your other boxed elements as a kind of jigsaw puzzle.

Box Model

From w3schools, the CSS box model is a box that wraps every HTML element. It consists of margins, borders, padding, and the actual content. You can adjust these properties with CSS and your boxed element will adjust accordingly. For example: 

              
              h1.blog {
                color: maroon;
                padding-left: 10px;
                font-weight: 600;
              }
