Personal Project Reflection

2016-05-21

Goals for the week

My main goal this week was to learn more of the language Go, and to write a simple server in Go. For my personal project I wanted to create a basic functioning webpage that showed a recipe to a user based on input like ingredients, course etc. Goals:

  • 1. Have the server working with an API.
  • 2. Be able to get data from the API on the front end
  • 3. Have a basic frontend worked out.
  • Stretch: Deploy the Go app to Heroku.

Reflection

I'm fairly happy with what I got done this week. I managed to tick off all my goals including my stretch goal. In hindsight, my stretch goal was a littly easy, but I wasn't sure what deployment would be like, and even if I'd get to the stage of deployment at all since I'd have to learn the basics of the language I was working with. The app is live at https://mangiamo.herokuapp.com/. By the way, mangiamo means "let's eat" in Italian.

The API I used is called food2fork. The most difficult part of the project was definitely finding the right API, and although food2fork isn't perfect, it has the basic functionality I want (mainly by having a free plan of 500 requests per day). At first I had some issues because food2fork doesn't support CORS, which meant that my client-side Javascript couldn't make requests to it (I tried using jsonp to get around this to no avail). But the simple solution was to make the requests to food2fork on the server-side, and then the client can make requests to my server instead of the food2fork one.

It was very satisfying from a technical knowledge standpoint to get a basic backend running in a language that I was previously unfamiliar with. Go was very fun to work with, and certainly something I'll try to continue with in the future. In fact, I enjoyed it so much that I also wrote a Go Version Manager for Windows. This helped me become more comfortable with the language, but there are still many things I want to try to make greater use of Go's features (interfaces, concurrency and more).

The app I made has the basic functionality I envisioned, but could look a lot nicer. I used Bootstrap and React for the frontend and didn't really spend a great deal of time on the client-side code. I could also add different user inputs, such as "course" and "vegetarian meals". The only tricky part about this is that the API I'm using supports just a basic keyword search, so I'd probably have to swap out this API for another one with more features (which could cost money...). There are a few more metrics from the current API, one being "social ranking" which I'm not quite sure how food2fork measures, but I could display this data to the user as well.

@Daniel Kermode
