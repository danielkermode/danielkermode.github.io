Javascript Refreshener

2016-05-12

Data Types

Javascript has seven distinct data types, six of which are primitives and one is a new ES6 feature (not available in all browsers). They are:

  • Boolean a logical entity, either true or false.
  • Null nothing. Not to be confused with a number 0 or an empty string ''. It is a "falsey" value.
  • Undefined a value that has not yet been assigned. Also "falsey".
  • Number in Javascript, numbers are always double precision 64-bit. There is no integer type (unlike many other languages). Number type can also have the values +Infinity, -Infinity and NaN. The last one is a bit mind-boggling considering NaN stands for "Not a Number", but life goes on.
  • String In Javascript, strings are immutable, and can only be "changed" by creating a new string with an operation such as +. Each string is a set of elements which are 16-bit unsigned integer values. Strings are used to represent textual data. Make sure you only use them for textual data, and not for data that could be better represented with something like an Object or a Number.
  • Symbol Symbols are a new ES6 feature and are unique and immutable. They can be used as Object keys.
  • Object These are best understood in Javascript as a collection of properties. Properties are identified as a key value (either a String or a Symbol). They can store any primitive value, including other objects. Functions are regular objects with the additional capability of being callable. Other more complex structures such as Arrays are also built on top of the base Object type.
Note: I borrowed all this from the MDN docs.

Specifically Arrays

As mentioned above, Arrays are a type of object built on top of the normal Object type. Create them literally with square brackets, eg. [1, 2, 3]. I've gone over some array methods and how to index an array in previous blogs, but for full details check out the MDN docs for Array. Arrays have a lot of built-in methods that are useful. For example .reverse() will do what you'd expect and put the elements in the array back-to-front.
